PostOffice Processing (Mail.dat Generation)

Complete SOP for generating Mail.dat files and preparing mailings for USPS submission.

Overview

This SOP details all steps to generate Mail.dat files for first-class letters and flats, verify reports, manage trays and pallets, upload to MD5, and coordinate with Presort EMS.

Important: Mail.dat files submitted to the Postal Office expire after 30 days. The postal office closes the month on the 10th of the following month. Ensure files are submitted on time.

1. Mailer Scorecard Check

  1. Open Postal Gateway and check the Mailer Scorecard.
  2. Note: Departmental Presort can be performed using BCC if required.

2. NetSort – First Class Letters

  1. Select First Class (top right) for yesterday’s letters.
  2. Run Duplicate Bar Code Check.
  3. Generate tray tags:
    • Select Scheme → All.
    • Set printer as Virtual Printer.
  4. Go to Reports → Report Batch:
    • Select yesterday’s job.
    • Ensure letters template matches the letters being processed.
    • Leave printer setting alone.
  5. Print the USPS Postage Summary Report:
    • Verify total pieces and mail types.
    • Metered, permit, and stamped mail for First Class (e.g., $0.25 for presort stamps).
  6. Export the Qualified Report and download it.
  7. Check Customer Mail Details Report for spelling or data issues in the first column.
  8. Container / Transfer Management:
    • View/Edit Trays → View Missing (confirm total trays for tags).
    • View/Edit Pallets → View Missing (confirm 18 pallets, print 2 copies for placards).
    • Click Manually Scan All.
  9. Mail.dat Transfer:
    • Move Update Method → Multiple.
    • Select Generate Full Service Complete Mail.dat.

3. NetSort – First Class Flats

  1. Switch to First Class Flats in the top right menu.
  2. Select Mailings → Today’s Date for flats, then click Save.
  3. Check duplicates:
    • Operations → USPS Duplicate Barcode Checks → Display MID Sequence Data.
  4. Tray Tags:
    • Typically skipped unless manually required.
  5. Print pallet placard tags:
    • USPS → Container Tags → Pallet from Analysis.
    • 8x11 format, select all, then Print → Cancel (adds placards to mail for tracking).
    • CSM file contains all containers.
  6. Reports → Mailing:
    • Select current flat job.
    • Report batch: 1st class reports (only vary for marketing mailings).
    • Print 3600-PFCM (Metered Postage Affixed).
    • Verify USPS Customer Mail Detail report:
      • Rate affixed should be $1.51.
      • No misspelled client names in the profile column.
  7. Container Management:
    • View/Edit Trays → View Missing.
    • View/Edit Pallets → View Missing.
    • Click Manually Scan All.
  8. Mail.dat Transfer:
    • Mailing → Today’s Flats
    • Mail Type → Tray-based First Pass
    • Move Update Method → Multiple
    • Generate Full Service Complete Mail.dat

4. MD5 Upload

  1. Upload both Mail.dat files (letters and flats) into MD5.
  2. While uploading, prepare the next day’s mailings in NetSort (APEX and Flexi machines).

5. Reconciliation

  1. Walk over to Dan’s office for VAR reports and Permit 123.xlsx reconciliation.
  2. Check USPS PostalOne Service Transaction History report.
  3. Log refunds and updates in the spreadsheet to keep account balances accurate.
  4. Move Mail.dat zip and report from USB to JLS Outbound O:\ drive for EMS consumption.

6. PreSort EMS Preparation

  1. Create mailings for tomorrow:
    • First Class Letters
    • Department of Labor & State of Rhode Island flats
  2. Ensure mailings match NetSort creation and barcodes generate properly.
  3. Receive transmittal slips from Eloy and create corresponding records in EMS QC Receiving.
  4. At 10:00 AM, Kourgas system pulls everything from hot folder into PreSort EMS.
  5. Monitor for errors; resolve any issues before continuing mail processing.

Notes

  • 150 pieces are required to reach a 5-digit mail count for processing.
  • Early analysis in Korber DB5 cannot be performed until sufficient pieces are loaded.
